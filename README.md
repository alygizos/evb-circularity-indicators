# EV Battery Circularity Indicators — NMC811 and LFP Cases

This repository contains two self-contained analysis packages for evaluating circularity indicators for electric vehicle batteries:

* **NMC811_codes_box** — NMC811 EV battery case
* **LFP_codes_box** — LFP EV battery case

Each package computes five circularity indicators:

* **PCI** — Product Circularity Indicator
* **CI** — Circularity Indicator
* **CEI** — Circular Economy Indicator
* **ECPI** — Energy Circularity Performance Indicator
* **M / R / E** — Composite Material, Resource, and Energy indicators

Both packages are designed to run independently. No external data files are required at runtime because the necessary input CSVs are already included in each folder.

---

## Requirements

Python 3.12 is recommended.

Install the required packages using either:

```bash
pip install -r requirements.txt
```

or manually:

```bash
pip install numpy pandas matplotlib seaborn openpyxl SALib
```

For headless or batch execution, set:

```bash
MPLBACKEND=Agg
```

This avoids GUI figure windows from blocking script execution.

## Main Code Folders

Each chemistry folder contains the same main indicator structure:

```text
Code/PCI/      PCI indicator script and input CSVs
Code/CI/       CI indicator script and input CSVs
Code/CEI/      CEI indicator script and input CSVs
Code/ECPI/     ECPI indicator script and input CSVs
Code/MRE/      Composite M / R / E indicator script and input CSVs
```

Each indicator script reads the input CSVs located in its own folder and writes its result to:

```text
Code/cumulative_results_indicators.csv
```

The compiled comparison figure is generated by:

```text
Code/compiled_results_indicator.py
```

## Running the NMC811 Case

From inside the `NMC811_codes_box/Code/` directory, run:

```bash
cd PCI  && python pci_indicator_v2.py
cd ../CI     && python ci_indicator.py
cd ../CEI    && python cei_indicator.py
cd ../ECPI   && python ecpi_indicator.py
cd ../MRE    && python mre_indicator.py
cd ..        && python compiled_results_indicator.py
```

Important: **MRE should be run last** in a batch run.
The PCI, CI, CEI, and ECPI scripts update the cumulative CSV for the four individual indicators, while MRE adds the composite **M, R, and E** results.

## Running the LFP Base Case

The LFP base case represents the **hydrothermal cathode route with 90% collection efficiency**.

From inside the `LFP_codes_box/Code/` directory, run:

```bash
for d in CEI/cei_indicator.py CI/ci_indicator.py ECPI/ecpi_indicator.py \
         PCI/pci_indicator_v2.py MRE/mre_indicator.py; do
  ( cd "$(dirname $d)" && python "$(basename $d)" )
done

python compiled_results_indicator.py
```

This updates:

```text
Code/cumulative_results_indicators.csv
```

and generates the compiled LFP indicator comparison figure.

## Running the LFP Solid-State Route

The LFP package also includes a solid-state route analysis.

From inside:

```text
LFP_codes_box/Code/solid_state/
```

run:

```bash
python run_lfp_solid_independent.py
```

This runs the five indicators using the included solid-route input CSVs and writes:

```text
cumulative_results_indicators_solid.csv
NMC811_vs_LFP_comparison_independent_lfp.png
```

The solid-state workflow is independent and does **not** overwrite the hydrothermal base-case results.

Alternatively, the indicator scripts can be run manually with the environment variable:

```bash
LFP_ROUTE=solid
```

For PowerShell:

```powershell
$env:LFP_ROUTE="solid"
```

Unset it after running the solid-state case to return to the hydrothermal base case.

## Running Sensitivity Studies

Both chemistry folders include self-contained sensitivity study runners.

### NMC811 Sensitivity

From:

```text
NMC811_codes_box/Code/sensitivity/
```

run:

```bash
python nmc811_sensitivity_lite.py
```

This recomputes the S1, S2, and S3 sensitivity studies and writes the corresponding CSV tables and comparison figures.

To rebuild only the clean base case:

```bash
python nmc811_sensitivity_lite.py --restore-base
```

### LFP Sensitivity

From:

```text
LFP_codes_box/Code/sensitivity/
```

run:

```bash
python lfp_sensitivity_lite.py
```

This runs the LFP sensitivity studies:

* **S1** — recycling-energy variation
* **S2** — recovery-rate variation
* **S3** — collection-rate variation

and writes the sensitivity Excel files, comparison figures, and raw-data workbook.

To restore the base cumulative results and figures:

```bash
python lfp_sensitivity_lite.py --restore-base
```

## Data Collection Folders

Each chemistry package includes a:

```text
Data Collection - Code/
```

folder.

These folders contain source workbooks, constants, provenance files, and optional input-generation scripts. They are kept for transparency and documentation, but they are **not required** to run the main indicator scripts.

For NMC811, the data collection folder includes:

```text
build_nmc811_ecpi_inputs.py
```

which can regenerate the ECPI raw-input CSV.

For LFP, the data collection folder includes:

```text
lfp_greet_data.py
lfp_bom.json
indicator_input_values_lfp.xlsx
indicator_inputs_variables_lfp.xlsx
LFP_DATA_COLLECTION_AND_EXECUTION.md
```

The LFP data-generation scripts are not bundled in the box edition because the input CSVs they produce are already included.

## Notes

* Both boxes are self-contained and can be shared independently.
* Indicator scripts write plots and update the cumulative results CSV.
* The ECPI method uses the revamped raw-input / `eq2` approach, with GHG calculations handled inside `ecpi_indicator.py`.
* For LFP, iron and phosphorus are included in the input data but are not recovered in the hydrometallurgical recycling assumptions; their recovery is set to zero across the indicators.
* For LFP, `compute_ci_cullen` is inlined in `ci_indicator.py`, so no shared module dependency is required.
* Backup files such as `*_PRE_REVAMP.*` and `*_PRE_LITE.*` are pre-update versions and can be safely removed if they are no longer needed.
* For NMC811, see `CHANGES_vs_MSEC.md` for the detailed change log.
